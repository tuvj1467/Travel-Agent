# Superpowers 强制工作流规则
<CRITICAL>
你已加载全套 superpowers 开发技能，所有编码任务必须优先执行 using-superpowers 技能检测。
1. 任何新增功能、重构、Bug修复，不允许直接写代码；
2. 需求开发标准链路：brainstorming（确认设计）→ writing-plans → executing-plans / subagent开发 → TDD → code review → finish分支；
3. 报错调试必须使用 systematic-debugging，禁止盲目改代码；
4. 必须完整走完技能内所有检查清单，不得跳过步骤；
5. 小型单行修改可手动声明跳过完整流程，其余场景强制全流程。
</CRITICAL>
